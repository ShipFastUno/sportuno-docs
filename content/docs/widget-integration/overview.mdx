The Sportuno widget is an embeddable iframe that delivers a full tournament experience — browsing, enrollment, betting, leaderboards, and rebuy/add-on — inside your platform without requiring you to build any game UI yourself.

## Architecture

The integration has three layers that interact at runtime:

```text
Your Backend                    Sportuno API
    |                               |
    |  POST /v1/widget/sessions     |
    |  Authorization: Bearer sk_*   |
    |------------------------------>|
    |                               |
    |<-- { session_token,           |
    |       widget_url,             |
    |       expires_at,             |
    |       refresh_before }        |
    |                               |
Your Frontend                   Embed iframe (widget_url)
    |                               |
    |  window.Sportuno.init({       |
    |    playerSessionId,           |
    |    gameSessionId              |
    |  })                           |
    |                               |
    |<---- PostMessage events ----->|
    |  IFRAME_READY                 |
    |  SESSION_EXPIRED              |
    |  REQUEST_SHARE                |
```

### Components

| Component | Role |
|-----------|------|
| **Your backend** | Calls Sportuno with a secret key (`sk_*`) to create session tokens. Never runs in the browser. |
| **Your frontend** | Loads the JS SDK, calls `window.Sportuno.init()`, manages token refresh. |
| **Sportuno JS SDK** | Mounts the iframe, forwards session tokens via PostMessage, relays events back to your page. |
| **Embed iframe** | The full Sportuno tournament UI, served from the Sportuno CDN. Communicates only via PostMessage. |

### Security model

- **Secret keys** (`sk_*`) are used only server-to-server. They are never embedded in page HTML or JavaScript.
- **Public keys** (`pk_*`) are safe to expose in the browser. They are passed to `window.Sportuno.init()` as `apiKey` to identify your operator account for catalog reads.
- The iframe communicates with the parent page using targeted `postMessage` calls. Origins are validated on both sides.
- The iframe sandbox attribute restricts capabilities to: `allow-scripts allow-same-origin allow-forms allow-popups allow-top-navigation-by-user-activation`.

<Callout type="warn" title="Never expose your secret key">
  The `sk_*` key must only be used in server-side code. If it is ever exposed in client-side JavaScript or version control, rotate it immediately from the operator dashboard.
</Callout>

## Session flow

1. User logs in to your site
2. Your backend creates a widget session — `POST /v1/widget/sessions` with `sk_*` key
3. Response includes `session_token`, `widget_url`, `expires_at`, `refresh_before`
4. Pass `session_token` to the JS SDK via `window.Sportuno.init()`
5. Iframe loads and fires `IFRAME_READY` — SDK sends the token to the iframe
6. Player interacts with tournaments
7. Before `refresh_before`, call `window.Sportuno.updateSession()` with a fresh token
8. If the token expires before refresh, `SESSION_EXPIRED` fires and your `onSessionExpired` callback handles it
