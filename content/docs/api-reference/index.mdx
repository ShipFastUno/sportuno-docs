## Overview

The Sportuno API is organized around three authentication tiers:

### Tier 1: Catalog (pk_* key)
Read-only access to tournaments, events, markets, and leaderboards.

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/v1/tournaments` | List tournaments |
| GET | `/v1/tournaments/{id}` | Get tournament details |
| GET | `/v1/tournaments/{id}/events` | Get tournament events |
| GET | `/v1/events/{id}/markets` | Get event markets |
| GET | `/v1/markets/{id}/selections` | Get market selections |
| GET | `/v1/odds` | Bulk get odds |
| GET | `/v1/tournament-groups` | List tournament groups |
| GET | `/v1/tournaments/{id}/leaderboard` | Get tournament leaderboard |
| GET | `/v1/leaderboards` | List leaderboards |

### Tier 2: Player Actions (Widget JWT)
Player enrollment, betting, rebuy, add-on, and profile management.

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/v1/tournaments/{id}/enroll` | Enroll in tournament |
| GET | `/v1/tournaments/{id}/my-enrollment` | Check enrollment status |
| POST | `/v1/tournaments/{id}/rebuy` | Execute rebuy |
| GET | `/v1/tournaments/{id}/rebuy-status` | Check rebuy availability |
| POST | `/v1/tournaments/{id}/addon` | Execute add-on |
| GET | `/v1/tournaments/{id}/addon-status` | Check add-on availability |
| POST | `/v1/tournaments/{id}/bets` | Place bet(s) |
| GET | `/v1/tournaments/{id}/bets/history` | Get bet history |
| POST | `/v1/realtime/token` | Get Ably realtime token |
| GET | `/v1/profile` | Get player profile |
| PUT | `/v1/profile` | Update player profile |

### Tier 3: Server-to-Server (sk_* key)
Session management. Must be called from your backend — rejected if Origin header present.

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/v1/widget/sessions` | Create widget session |
| POST | `/v1/widget/sessions/refresh` | Refresh widget session |

## OpenAPI Specification

The full OpenAPI 3.1 specification is available at:

```
https://api-uat.sportuno.win/v1/.well-known/openapi.json
```

You can import this spec into tools like [Postman](https://www.postman.com/), [Insomnia](https://insomnia.rest/), or [Scalar](https://scalar.com/) for interactive exploration.
