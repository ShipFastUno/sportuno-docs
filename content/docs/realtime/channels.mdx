## Channel Naming Convention

All Sportuno Ably channels follow this pattern:

```
sportuno:{namespace}:{id}
```

Examples:
- `sportuno:tournament:tour_01JKQMVS` — a specific tournament
- `sportuno:market:mkt_01JKQMVS` — a specific market
- `sportuno:user:usr_01JKQMVS` — a specific player's private channel
- `sportuno:lobby` — the global tournament lobby (no ID suffix)

## Channel Reference

### Public Channels (accessible with `pk_*` or widget JWT)

| Channel Pattern | Description | Key Events |
|-----------------|-------------|------------|
| `sportuno:tournament:{id}` | Tournament-level updates | `status_changed`, `participant_count_changed`, `phase_changed` |
| `sportuno:market:{id}` | Market status and odds updates | `status_changed`, `odds_updated` |
| `sportuno:leaderboard:{id}` | Leaderboard position changes | `leaderboard.updated`, `position_changed` |
| `sportuno:event:{id}` | Sporting event updates | `status_changed`, `score_updated` |
| `sportuno:odds:{id}` | Odds-only channel (high frequency) | `odds_updated` |
| `sportuno:lobby` | Global tournament lobby | `tournament_added`, `tournament_removed`, `tournament_updated` |

### Authenticated Channels (widget JWT only)

| Channel Pattern | Description | Key Events |
|-----------------|-------------|------------|
| `sportuno:user:{user_id}` | Player notifications | `notification`, `balance_updated` |
| `sportuno:player:{player_id}` | Player game state | `enrollment_confirmed`, `bet_settled`, `rebuy_available` |
| `sportuno:ticker:{tournament_id}` | Live tournament event ticker | `tick`, `alert` |

<Callout type="info">
Channels prefixed `sportuno:user:*` and `sportuno:player:*` are scoped to the authenticated player. Sportuno issues tokens that only permit access to the requesting player's own channels — cross-player snooping is prevented at the capability level.
</Callout>

## Subscribing to Events

### TypeScript / JavaScript

```typescript
import Ably from 'ably';

const client = new Ably.Realtime({ token: sportunoToken });

// Subscribe to a specific market's status changes
const marketChannel = client.channels.get('sportuno:market:mkt_01JKQMVS');
marketChannel.subscribe('status_changed', (msg) => {
  const { market_id, status, timestamp } = msg.data;
  console.log(`Market ${market_id} is now ${status}`);
});

// Subscribe to leaderboard updates
const lbChannel = client.channels.get('sportuno:leaderboard:lb_01JKQMVS');
lbChannel.subscribe('leaderboard.updated', (msg) => {
  const { entries, updated_at } = msg.data;
  renderLeaderboard(entries);
});

// Subscribe to player-specific notifications (requires widget JWT)
const playerChannel = client.channels.get('sportuno:player:plr_01JKQMVS');
playerChannel.subscribe('bet_settled', (msg) => {
  const { bet_id, result, payout } = msg.data;
  showBetResult(bet_id, result, payout);
});
```

## Event Payload Schemas

### market:* — status_changed

```json
{
  "market_id": "mkt_01JKQMVS",
  "event_id": "evt_01JKQMVS",
  "status": "suspended",
  "timestamp": "2026-03-24T10:00:00Z"
}
```

| Status | Description |
|--------|-------------|
| `open` | Market is accepting bets |
| `suspended` | Market temporarily closed (e.g., goal scored) |
| `closed` | Market permanently closed for new bets |
| `settled` | Market result confirmed, bets resolved |
| `cancelled` | Market cancelled, bets voided |

### market:* — odds_updated

```json
{
  "market_id": "mkt_01JKQMVS",
  "selections": [
```

## High-Frequency Channels

The `sportuno:odds:{id}` channel publishes at very high frequency during live events — dozens of updates per minute per market. Subscribe to specific markets rather than wildcards to avoid unnecessary message volume.

<Callout type="warn">
Ably bills on message volume. If you are subscribing from custom code outside the widget, be selective about which channels you subscribe to. The widget's built-in subscription management handles this efficiently for in-widget use.
</Callout>

## Token Refresh

Ably tokens are short-lived. Configure your Ably client to request a new token before expiry:

```typescript
const client = new Ably.Realtime({
  authCallback: async (_tokenParams, callback) => {
    try {
      const resp = await fetch('/api/realtime-token'); // your backend fetches from Sportuno
      const { token } = await resp.json();
      callback(null, token);
    } catch (err) {
      callback(err as Error, null);
    }
  },
});
```

This pattern ensures the client automatically re-authenticates without dropping subscriptions.
